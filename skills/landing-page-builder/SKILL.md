# Landing Page Builder

## Purpose

Crear landing pages profesionales, claras y orientadas a conversión, con diseño responsive y copy accionable.

## When to use

- Para páginas de venta, captura de leads, servicios, productos, demos o campañas.
- Cuando el usuario pide una landing, home comercial o página de agencia.
- Para mejorar copy y estructura de conversión.

## When not to use

- Para dashboards o herramientas internas.
- Para documentación técnica.
- Para páginas de producto donde ya existe un diseño system rígido que debe seguirse sin cambios.

## Procedure

1. Define audiencia, oferta, dolor principal y acción deseada.
2. Estructura la página: hero, problema, solución, beneficios, servicios, prueba social, proceso, pricing, FAQ y CTA.
3. Escribe copy específico: titular claro, promesa verificable, beneficios concretos y objeciones respondidas.
4. Diseña responsive desde móvil y desktop, con jerarquía visual y estados accesibles.
5. Usa assets visuales relevantes para producto, servicio o resultado; evita decoración genérica.
6. Implementa secciones con semántica HTML, buen contraste, foco visible y performance razonable.
7. Incluye CTA persistente o repetido sin saturar.
8. Verifica layout en anchos pequeños y grandes, textos sin solaparse y carga sin errores.

## Output checklist

- Oferta y CTA claros en primer viewport.
- Secciones comerciales completas.
- Copy específico y escaneable.
- Diseño responsive.
- Accesibilidad básica revisada.
- Performance y SEO básico considerados.

## Common mistakes

- Crear una página bonita sin propuesta de valor.
- Usar hero genérico sin mostrar producto, resultado o contexto real.
- Hacer pricing o prueba social ficticia sin señalar que es mock.
- Ignorar móvil, contraste o largos de texto.
