# Client Demo Builder

## Purpose

Crear demos rápidas que muestren valor de forma concreta, visual y fácil de entender para un cliente.

## When to use

- Para prototipos comerciales, pruebas de concepto y demos de ventas.
- Cuando el usuario necesita impresionar o explicar una idea rápidamente.
- Para convertir una propuesta en un flujo visible.

## When not to use

- Para sistemas productivos completos.
- Para ocultar limitaciones técnicas como si ya estuvieran resueltas.
- Para demos con datos sensibles reales.

## Procedure

1. Define el caso de uso único que la demo debe probar.
2. Elige un flujo simple: entrada, acción principal, resultado y siguiente paso.
3. Usa datos mock realistas, etiquetados y seguros.
4. Diseña pantalla inicial con valor visible de inmediato.
5. Implementa interacciones suficientes para contar la historia, no todo el producto.
6. Incluye estados de loading, éxito, error o vacío si ayudan a la credibilidad.
7. Añade una explicación breve del valor y CTA comercial.
8. Verifica que la demo corre localmente y que el recorrido principal no se rompe.

## Output checklist

- Caso de uso concreto.
- Flujo corto y navegable.
- Datos mock realistas.
- Visual claro y profesional.
- CTA o siguiente paso.
- Instrucciones para ejecutar la demo.

## Common mistakes

- Intentar construir el producto completo.
- Usar lorem ipsum o datos irreales.
- Hacer demasiadas pantallas sin un momento de valor.
- No distinguir mock, simulado y funcional.
