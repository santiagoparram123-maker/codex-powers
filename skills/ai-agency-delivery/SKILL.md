# AI Agency Delivery

## Purpose

Estructurar entregables de una agencia de IA para que el cliente entienda valor, alcance, demo, implementación y handoff.

## When to use

- Para propuestas, diagnósticos, MVPs, automatizaciones o demos de agencia de IA.
- Cuando se necesita conectar problema de negocio con solución técnica.
- Para preparar handoff, documentación y siguientes pasos con cliente.

## When not to use

- Para investigación académica de IA.
- Para implementar modelos desde cero sin caso de negocio.
- Para prometer resultados no verificables o automatizaciones sin límites claros.

## Procedure

1. Define cliente, proceso actual, dolor, costo del problema y métrica de éxito.
2. Propón diagnóstico: flujo actual, tareas repetitivas, datos disponibles, riesgos y quick wins.
3. Diseña solución: demo, MVP, integración, automatización, humano en el loop y límites.
4. Divide entregables: propuesta técnica, demo navegable, MVP, documentación y handoff.
5. Incluye criterios de aceptación y métricas: tiempo ahorrado, reducción de errores, tasa de respuesta, conversión.
6. Documenta dependencias: APIs, credenciales, datos, permisos, compliance y mantenimiento.
7. Prepara narrativa de demo: problema, antes/después, flujo, valor y CTA.
8. Encadena skills según necesidad: landing, demo, docs o deploy.

## Output checklist

- Problema de negocio definido.
- Entregables claros y secuenciados.
- Alcance de MVP y no-alcance.
- Métricas de éxito.
- Riesgos y dependencias.
- Handoff y próximos pasos.

## Common mistakes

- Vender "IA" sin workflow concreto.
- No aclarar datos, permisos o integraciones.
- Crear una demo desconectada del ROI.
- Omitir límites, fallbacks y revisión humana.
