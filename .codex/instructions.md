# Codex Instructions

Load `CODEX.md` as the primary operating guide for this repository.

When a task maps to a skill in `skills/`, read that skill's `SKILL.md` before acting. Use `examples/`, `checklists/`, and `templates/` only when they are relevant to the task.

Default workflow:

1. Inspect the project.
2. Choose the relevant skill.
3. Plan the smallest useful change.
4. Implement.
5. Verify.
6. Summarize changes and risks.
