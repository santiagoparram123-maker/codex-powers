# Methodology

Codex Powers follows a simple delivery loop:

1. Understand the project.
2. Clarify the request.
3. Plan the change.
4. Implement in small steps.
5. Verify with evidence.
6. Review for security, UX, copy, and delivery quality.
7. Summarize the result.

## Principles

- Evidence beats memory. Inspect files, scripts, tests, and logs.
- Small changes reduce review risk.
- Tests are most valuable when they protect behavior that matters.
- Security and UX are part of delivery, not optional polish.
- Documentation should help the next person make a safe change.

## Skill Chaining

Use one skill when the task is narrow. Chain skills when the work crosses phases.

Example:

```text
project-intake -> requirements-clarifier -> implementation-planner -> tdd-implementer -> code-reviewer -> deployment-checklist
```

Do not chain skills just to be formal. Use only the skills that improve the outcome.
